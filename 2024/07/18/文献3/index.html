A Threshold Proxy Re-Encryption Scheme for Secure IoT Data Sharing Based on Blockchain

WHY：

物联网设备存储容量有限，数据需要经过加密后上传至第三方进行存储。有必要建立有效的访问控制机制以及代理重加密机制分担数据所有者计算压力。同时为了消除对于第三方中心服务提供者的依赖（代理服务节点），引入阈值代理重加密，并结合区块链的共识，实现不可篡改性与透明性。

HOW：

基于阈值的代理重加密机制：传统的代理重加密技术通常涉及单一代理或非分布式的重加密策略。该文献提出了一种基于阈值的模型，其中重加密操作需要多个节点（即达到一定数量的共识节点）共同参与和完成，这不仅增强了重加密过程的安全性，也减少了单点故障的风险。
集成区块链的共识机制：通过利用区块链的共识机制来验证和记录每一次的代理重加密操作和密钥管理活动，确保了整个过程的不可篡改性和透明度。这种方法提高了系统的信任度，并且通过分布式账本技术，增强了跨域数据共享的效率和可行性。因为共识的存在，在此方案中，密钥不是由单一的中心化实体控制，而是分散存储和管理，需要多个授权节点协同工作才能完成重加密任务。这种分散的密钥管理方法降低了密钥泄露或滥用的风险。

阈值代理重加密:

将重新加密委托给多个代理节点。该机制可以实现多个代理之间的信任分割，避免了代理与被委托之间的勾结。同时也解决了单个代理容易收到拒绝服务攻击的问题。此外，单个代理可能存在性能瓶颈和单点故障。

Shamir秘密共享：

将秘密分成N个参与者保管。只有当至少t个参与者在场时，才能恢复秘密。

拉格朗日插值：

对于给定的n+1个点（x0，y0），（x1，y1），…，（xn，yn）对应于它们次数都不超过n的拉格朗日多项式L只有一个。如果计入次数更高的多项式，则有无穷个，因为所有于L相差λ（x-x0）（x-x1）…（x-xn）的多项式都满足条件。

将区块链共识算法机制和阈值代理加密计算相结合:

数据代理将重加密密钥拆分并发送给多个代理节点，这些代理节点可以从区块链网络中的共识节点中选择。这些节点在收到委托方的重加密请求后，根据收到的重加密密钥计算并生成相应的“重加密份额“，将请求和密文转换记录在账本上。将共识节点的哈希问题替换为使用重加密密钥转换密文的过程，并根据其他节点的验证结果生成和写入新的区块。

POW（工作量证明）：简单来说就是所有矿工公平竞争一个区块的挖矿权。他们对候选的区块进行Hash运算，当算出对应结果后，就可以得到挖矿权，并全网广播，其他节点会对运算结果进行验证，若超过一半的节点对该结果表示赞同，则出块，对应矿工享受出块周期的收益。

为什么采用Hash呢？

Hash具有难以计算，但易于验证的性质。其他节点可以轻松验证结果的对错。

将哈希问题替换为使用重加密密钥转换密文，对重加密过后的密文Cbi签名，其他节点对签名进行验证，签名验证的过程也是属于难计算但易验证。

四个实体：

DO（Data owners）:拥有个人数据的所有权，这些数据需要经过对称加密后上传至SSP中存储。DO用自己的公钥对数据的元数据信息进行非对称加密，发布到区块链上（包括数据摘要、数据存储位置、数据密文的签名和对称密钥）在收到数据用户的访问请求后，验证访问权限，根据DU的公钥和自己的私钥生成重加密密钥。重加密密钥被发送至区块链的共识节点进行密文转换和共享。

DU（Data users）：包括一系列需要使用物联网设备数据的机构。DU通过浏览区块链上的数据摘要来搜索感兴趣的内容。通过基于属性访问控制和可搜索加密算法的智能合约实现。DU在获得授权后，获得重新加密的元数据密文，DU可以使用自己的私钥对重新加密的密文进行解密，获得元数据信息。再向SSP发送访问请求，获取加密数据解密

区块链网络：区块链共识节点扮演代理服务器的角色。采用阈值代理重加密方案。DO将多个重加密密钥共享分发给区块链网络中的多个节点。共识节点使用相应的重新加密共享对原始密文进行重新加密。转换后的元数据密文被收集并打包成新的块。改过程实现依赖设计的区块链共识算法。

SSP（Storage Service Providers）：物联网收集的数据经过加密后上传至第三方服务器进行存储和共享。如IPFS。

阈值代理重加密过程：
1.System setup（1^λ^，U）

生成随机数g，乘法循环群G1，G2，Z=e（g，g），四个哈希函数。

2.Key Generation（PP）—>（SKDO，PKDO，SKDU，PKDU）

获得用户私钥和公钥。

SKDO=a，SKDU=b,PKDO=g^a^,PKDU=g^b^

3.Data Encryption(DEK,Data)->CT

通过对称加密获得密文CT

4.Metadata Encry（PP，PKDO,M）—>CTM

CTM=（g^ak^，MZ^k^）,其中Z=e（g，g），k是一个随机数（防止攻击者通过比较重复的加密输出判断原始元数据的内容）

5.Re-encry Key Generation

由所有者私钥和访问者公钥生成

RKDO->DU=g^b/a^（需要隐藏的共享秘密s）

阈值代理重加密与共识结合：

1.每个共识节点根据接收到的重加密密钥，对元数据密文进行重加密转换。共识节点先再区块链账本上找到对应的原始密文CTM，再使用重加密密钥rk^i^ O->U执行转换过程。

2.在限定实际内完成密文转换Cbi，并进行签名，广播到区块链网络中，每个共识节点收集并验证接收到的重新加密的密文Cbi。当一个共识节点率先收集t个经过验证的重新加密的密文时；向区块链网络发送成为领导节点的请求，并启动一轮投票。在收到此请求后，其他共识节点验证t重新加密的密文正确性。若超过一半节点支持该结果，则发送此请求的节点成为领导节点，获得共识过程记录权。

3.领导节点将重新加密的密文Cbi合并为新的密文CTM‘,并且添加到区块链分类账上，向网络广播。其他节点完成共识确认并更新账本，将转换后的元数据密文CTM‘写入更新块。

为什么需要共识：

没有一个可信的第三方，无法将碎片合并到一起，所以需要节点之间互相通信，所以需要共识。

思考：

这篇文章将阈值代理重加密和共识结合，共识类型更偏向于POW，如果仿照之前读过的一篇论文（基于RS编码的联盟链存储方案），其中的数据分片于编码过程有点类似于文章中的代理重加密密钥碎片共享，按照PBFT的过程来进行共识。进一步的，可以参考（基于分组编码的联盟链存储方案），参考在RS编码的基础上实现分组编码，也可以考虑将代理重加密密钥进行分组共享。

如果加入监管的话，有两种办法：（1）加入第三方监管组织。（2）节点内部行为监管（融合在共识过程中）：每个节点都有信誉值。节点可以通过对数字签名的验证来发现其他节点的恶意行为，当发现不合法是，广播detected消息。收到F+1个消息后，更新数组中节点的信誉值。
